# UKI contract freeze checklist

Estado: not frozen.
Issue: #134 `UKI-090.1`.
Fecha: 2026-05-13.

## Purpose

This checklist defines what must be true before the UKI sale contracts are considered frozen for launch.

It does not freeze the contracts by itself. Freeze only happens when final addresses, verification links, ABI hashes, role state and QA evidence are recorded from the real BSC testnet/mainnet deployment.

## Freeze outputs

The final freeze PR must include:

- final contract addresses for `UKIToken`, `VestingVault` and `Presale`,
- ASM token address and treasury address,
- BSC chain id and BscScan base URL,
- BscScan verification links for every deployed contract,
- exported dapp ABIs committed under `dapp/src/lib/contracts/abis`,
- freeze manifest generated by `pnpm --filter @cukies/contracts freeze:manifest`,
- final owner/admin/role matrix,
- validation commands and outputs,
- explicit list of approved launch parameters.

| Vault funding | `VestingVault` funded with at least `totalUkiForSale`. | Pending deploy. |
| Presale role grant | `Presale` has `VESTING_MANAGER_ROLE`. | Pending deploy. |
| Dapp config | `NEXT_PUBLIC_*` env values match deployed addresses. | Pending deploy. |
| ABI versioning | ABI export committed and hash recorded in manifest. | Pending final deploy. |
| External review | Audit or independent review complete. | Pending. |

## Manifest generation

Compile first so artifacts are current:

```bash
pnpm --filter @cukies/contracts compile
```

Generate a manifest using final env values:

```bash
NEXT_PUBLIC_UKI_CHAIN_ID=97 \
NEXT_PUBLIC_BSCSCAN_BASE_URL=https://testnet.bscscan.com \
NEXT_PUBLIC_ASM_TOKEN_ADDRESS=0x... \
NEXT_PUBLIC_UKI_TOKEN_ADDRESS=0x... \
NEXT_PUBLIC_UKI_VESTING_VAULT_ADDRESS=0x... \
NEXT_PUBLIC_UKI_PRESALE_ADDRESS=0x... \
SALE_OWNER_ADDRESS=0x... \
SALE_TREASURY_ADDRESS=0x... \
pnpm --filter @cukies/contracts freeze:manifest
```

For mainnet freeze, use chain id `56` and `https://bscscan.com`.

## Freeze rule

After freeze, any contract, ABI, address or launch parameter change requires:

1. a new issue linked to #134,
2. a new PR,
3. rerun of compile, tests, coverage, static analysis and testnet flow,
4. updated manifest,
5. explicit approval before mainnet use.

## Current blockers

- Final multisig/admin owner address is not recorded.
- BSC testnet deployment evidence is missing.
- BscScan verification evidence is missing.
- Slither/static analysis evidence is missing.
- External audit or independent security review is missing.
- `UKIStaking` and `RewardsDistributor` are not part of the current sale freeze and need separate freeze gates when implemented.
