# UKI contract freeze checklist

Estado: not frozen.
Issue: #134 `UKI-090.1`.
Fecha: 2026-05-13.

## Purpose

This checklist defines what must be true before the UKI sale contracts are considered frozen for launch.

It does not freeze the contracts by itself. Freeze only happens when final addresses, verification links, ABI hashes, role state and QA evidence are recorded from the real BSC testnet/mainnet deployment.

## Freeze outputs

The final freeze PR must include:

- final contract addresses for `UKIToken`, `VestingVault` and `Presale`,
- ASM token address and treasury address,
- BSC chain id and BscScan base URL,
- launch Safe address, signer list and threshold,
- deployer address and confirmation that it has no unnecessary owner/admin/manager powers,
- BscScan verification links for every deployed contract,
- exported dapp ABIs committed under `dapp/src/lib/contracts/abis`,
- freeze manifest generated by `pnpm --filter @cukies/contracts freeze:manifest`,
- final owner/admin/role matrix,
- validation commands and outputs,
- explicit list of approved launch parameters.
- TGE vesting policy: whether `VestingVault.presaleVestingConfigFrozen()` is still false before TGE or true before claims.

## Environment matrix

Do not mix dev, testnet and production addresses. Each freeze or deploy review must name the target environment explicitly.

| Environment | Chain id | ASM source | Expected ASM address |
| --- | --- | --- | --- |
| Local dev | `31337` | `MockERC20` from Hardhat tests/simulations only. | Local deployment address, never reused outside local. |
| BSC testnet | `97` | Test ASM / `tASM`. | `0xf93dd40Bf8bD8dDf7C785AA87dc13C3c3FeB6c8C` |
| BSC mainnet | `56` | Production ASM / `CONCILIUM`. | `0x40af8fd127dcd302d7ffa6f37cf5a002e54ac68c` |

The deploy script rejects non-local deploys when `ASM_TOKEN_ADDRESS` does not match the approved address for the target chain id. The approved ASM token is confirmed as standard ERC-20 with no fee-on-transfer/rebase behavior.

Compile first so artifacts are current:

```bash

```

Generate a manifest using final env values:


## Freeze rule

After final contract freeze, any contract, ABI or address change requires:

1. a new issue linked to #134,
2. a new PR,
3. rerun of compile, tests, coverage, static analysis and testnet flow,
4. updated manifest,
5. explicit approval before mainnet use.

Sale parameters are not contract-frozen by design. Live changes to `treasury`, `saleStart`, `saleEnd`, `minAsmPerPurchase`, `totalUkiForSale` or `ukiPerAsm` require Safe approval, reason, calldata review and tx hash in the operations log.
