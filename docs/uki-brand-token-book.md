# Cukies World / UKI - Brand & Token Book

Estado: borrador v0.1 para validacion.
Uso: fuente de verdad para branding, design tokens, UX visual, copy base e imagenes generadas.
Principio: los assets actuales inspiran el ADN Cukies, pero no obligan a copiar el estilo antiguo.

## 1. Tesis de marca

Cukies World no debe reposicionarse como "otra preventa crypto".

Debe reposicionarse como:

> una coleccion NFT con historia que evoluciona hacia una economia de juegos propia, donde UKI es el activo que conecta preventa, juegos, creditos, pools, recompensas y futuras experiencias.

La marca tiene que resolver una tension:

- Cukies es cute, colorido, coleccionable y reconocible.
- UKI necesita confianza, claridad economica y sensacion de producto serio.

La solucion visual no es abandonar los Cukies. Es elevarlos.

## 2. Direccion creativa recomendada

### Nombre de direccion

**Arcade Vault Adventure**

### Idea central

Un mundo de criaturas coleccionables entra en una nueva fase: una tesoreria jugable, conectada a BNB Smart Chain, donde el token UKI abre la economia de juegos.

### Formula visual

```text
Cukies cute + aventura de mundo + vault/token UI + dashboard economico oscuro
```

### Que conserva del ADN actual

- Cukies con contorno fuerte, ojos grandes y siluetas reconocibles.
- Color de rarezas y sensacion coleccionable.
- Mapas, rutas, mundo, señales, aventura.
- UKI como moneda dorada y objeto de recompensa.
- Tono cercano y de comunidad.

### Que moderniza

- La web ya no debe parecer una pagina NFT de 2021.
- La preventa necesita jerarquia, datos claros y confianza.
- Los paneles economicos deben parecer producto real, no infografia.
- Los assets cartoon deben usarse como acentos narrativos, no como toda la interfaz.

### Que descarta

- Secciones que parecen diapositivas.
- Bloques explicativos sueltos sin flujo de landing.
- Exceso de glow crypto.
- Gradientes morado/azul genericos.
- Promesas de rentabilidad.
- Estetica casino o meme coin.
- UI de madera/cartel para acciones financieras.

## 3. Arquitectura de marca

### Marca madre

**Cukies World**

Rol: universo, comunidad, coleccion NFT, juegos y continuidad historica.

### Token

**UKI**

Rol: activo economico de la nueva etapa.

No debe verse como un token desconectado. Siempre debe aparecer vinculado a:

- juegos,
- creditos,
- recompensas,
- Cukie Master,
- economia multi-juego,
- BNB Smart Chain.

### Producto / capa economica

**Cukies Game Economy**

Rol: sistema donde viven juegos, creditos, pools, rankings y recompensas.

Este nombre evita que Treasure Hunt parezca la economia completa.

### Primer juego

**Treasure Hunt**

Rol: primer caso de uso jugable de UKI.

Debe comunicarse como "primer juego conectado a la economia", no como el producto total.

### Figura de estatus

**Cukie Master**

Rol: participante avanzado de la economia.

Importante: Cukie Master y staking son fase posterior a la preventa. En la home de lanzamiento pueden aparecer como "coming next" o "after presale".

## 4. Orden de comunicacion para lanzamiento

### Fase 1: Preventa UKI

Objetivo principal: captar interes y compras de UKI.

Debe verse primero:

1. UKI Presale.
2. Precio / duracion / red.
3. Compra con ASM.
4. Vesting.
5. Liquidez.
6. Por que UKI existe.

### Fase 2: Utilidad post-venta

Objetivo: explicar por que UKI importa despues de comprar.

Debe verse despues:

1. Cukie Master.
2. Creditos diarios.
3. Pools.
4. Ranking semanal.
5. Treasure Hunt como primer juego.
6. Futuras experiencias.

### Fase 3: Operacion de economia

Objetivo: mostrar producto real cuando se active.

Debe vivir en la dapp, no necesariamente en la landing:

1. Dashboard wallet.
2. Staking UKI.
3. Pool de Cukies.
4. Pool de creditos.
5. Arena ranking.
6. Rewards claim.

## 5. Personalidad verbal

### Rasgos

- Claro.
- Directo.
- De juego, pero no infantil.
- Tecnico cuando habla de token.
- Cercano cuando habla de comunidad.
- Transparente cuando habla de vesting, liquidez y reglas.

### Tono recomendado

```text
Una comunidad NFT conocida entra en su fase jugable.
UKI no es solo un ticker: es la pieza que conecta Cukies, juegos y recompensas.
```

### Frases base

- "UKI abre la nueva economia de Cukies World."
- "La preventa es el primer paso. La utilidad empieza despues."
- "Treasure Hunt sera el primer juego conectado a UKI."
- "Cukie Master llega despues de la venta, con cupos, creditos y pools."
- "Todo el sistema UKI opera en BNB Smart Chain."
- "Los NFTs siguen siendo parte del mundo Cukies, ahora conectados a una economia jugable."

### CTAs

Primarios:

- "Join UKI Presale"
- "Enter UKI Presale"
- "Comprar UKI"

Secundarios:

- "Ver detalles de la venta"
- "Leer tokenomics"
- "Explorar Cukie Master"
- "Ver como funciona la economia"

### Lenguaje sensible

Evitar hasta validacion legal:

- "ingresos pasivos"
- "retorno garantizado"
- "rentabilidad"
- "yield"
- "APR"
- "garantizado"
- "beneficio asegurado"

Usar en su lugar:

- "recompensas"
- "asignaciones"
- "programa de recompensas"
- "participacion en pools"
- "claim"
- "distribucion segun reglas"

## 6. Sistema visual

### Direccion general

La interfaz debe sentirse como una **boveda arcade**:

- Oscura.
- Profunda.
- Con acentos turquesa.
- Con UKI en oro.
- Con Cukies como presencia viva.
- Con paneles de producto reales.

No debe sentirse como:

- pitch deck,
- whitepaper convertido en web,
- landing SaaS generica,
- meme coin,
- casino,
- web infantil.

### Uso de ilustracion

Los Cukies deben aparecer como:

- personajes de apoyo,
- sellos de rareza,
- pequenas escenas,
- stickers de estado,
- objetos de juego,
- momentos de recompensa.

No deben ocupar todos los paneles ni competir con la venta del token.

### Uso de UKI

UKI debe aparecer como:

- moneda principal,
- ficha de compra,
- elemento de boveda,
- unidad de reward,
- token en interfaces de compra/vesting/claim.

UKI debe tener mas presencia que cualquier otro simbolo en la home de preventa.

## 8. Typography tokens

### Roles

Logo:

- Usar logo actual o evolucionado, nunca reemplazarlo por texto generico.

Display:

- Uso: hero, grandes titulares, momentos de marca.
- Direccion: grotesk expresiva, compacta, con caracter.
- Candidatas: `Space Grotesk`, `Cabinet Grotesk`, `Clash Display`.

UI/body:

- Uso: textos, formularios, paneles, dashboard.
- Direccion: sans limpia, legible, no infantil.
- Candidatas: `Geist Sans`, `Satoshi`, `Space Grotesk`.

Numbers/technical:

- Uso: precio, vesting, balances, claims, ranking.
- Direccion: mono o semi-mono.
- Candidatas: `JetBrains Mono`, `Geist Mono`.

### Type scale

## 13. Landing structure recomendada

La landing no debe explicar todo a la vez. Debe vender la primera accion y abrir la puerta al sistema.

### 1. Hero - UKI Presale

Objetivo: que el usuario entienda que lo primero es la venta.

Contenido:

- UKI Presale.
- BNB Smart Chain.
- compra con ASM.
- precio/duracion/vesting.
- CTA principal.

Visual:

- modulo de compra real.
- UKI coin/vault.
- Cukie como apoyo, no protagonista principal.

### 2. Sale Mechanics

Objetivo: reducir dudas sobre como participar.

Contenido:

- Connect wallet.
- Approve ASM.
- Buy UKI.
- Track vesting.

### 3. Token Trust

Objetivo: explicar liquidez, vesting y estructura.

Contenido:

- ASM raised -> UKI liquidity.
- liquidity lock/burn.
- buyer vesting.
- team vesting.
- rewards program.

### 4. Why UKI Exists

Objetivo: evitar que parezca solo preventa.

Contenido:

- UKI conecta juegos, creditos, pools, rewards y futuras experiencias.

### 5. Cukies World Legacy

Objetivo: usar la historia 2021 sin parecer antiguo.

Contenido:

- NFT collection launched 2021.

### 6. After Presale: Cukie Master

Objetivo: explicar utilidad futura sin distraer del CTA principal.

Contenido:

- staking UKI,
- Cukie points,
- creditos diarios,
- cupos,

### 7. First Game: Treasure Hunt

Objetivo: mostrar primer uso jugable.

Contenido:

- legal-safe rewards.

## 14. Reglas para que no parezca una presentacion

Una seccion web debe:

- tener una accion o decision clara,
- permitir escaneo,
- tener continuidad visual con la anterior,
- usar menos texto del que tendria una diapositiva,
- mostrar producto o estado real,
- no depender de flechas gigantes o diagramas explicativos.

Si una seccion parece una slide, corregir:

- reducir texto,
- cambiar diagrama por UI real,
- mover detalles a FAQ,
- usar un modulo interactivo,
- separar fase actual de fase futura.

## 15. Imagenes generadas: direccion y restricciones

### Antes de generar

Cada prompt debe declarar:

- objetivo de la pantalla,
- fase: preventa / post-venta / juego / rewards,
- que debe dominar visualmente,
- que NO debe aparecer,
- si se permite usar Cukies como protagonistas o solo como apoyo.

### Estilo de prompts

Usar:

- premium game economy,
- dark teal vault UI,
- cute Cukies elevated into a serious token launch,
- UKI gold as economic focus,
- real landing/product interface,
- not a presentation slide,
- not a crypto deck.

Evitar:

- infographic,
- pitch deck,
- slide,
- generic crypto landing,
- casino,
- meme coin,
- childish cartoon website.
